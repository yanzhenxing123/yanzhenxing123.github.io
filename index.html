StatsD

StatsD 狭义上来讲，其实就是一个监听 UDP(Default)/TCP 的守护程序

基本分为了三步，

  1. 根据简单的协议收集 StatsD 客户端发送来的数据;
  2. 解析数据并进行聚合；
  3. 定时推送给后端，如 Graphite 和 InfluxDB 等，并通过 Grafana 等展示

github链接: https://github.com/statsd/statsd/blob/master/docs/metric_types.md

StatsD metrics

格式：<metric_name/bucket>:<value>|<type>[|@sample_rate]

指标类型： counter、timer、set、guage

counter

timing

计时器，可以计算平均值、最大值、最小值等

gauge

标量是任何可测量的一维变量，如内存、人的身高、体重等。

如果该值在下次刷新前没有更新，那么就会发送上次的值。也可以在值前添加正负号，用于表示对值的修改，而非设置。这也就意味着，不能直接设置负值，需要先设置为 0 才可以。

set

记录flush期间不重复的值。

可以针对某一个集合进行统计，统计总共出现了多少种类的值。

QPS（Query Per Second）是用来衡量服务性能的一个重要指标，它的定义：对一个特定的查询服务器在规定时间内所处理流量的多少，计算公式：QPS = 并发量 / 平均响应时间。

通常 QPS 用来衡量服务器性能，不断的为增加它的数量而优化改进，通过多线程、增加负载、甚至提高代码质量、算法优化等方式。当然优化性能是有上限的，我们需要在性能和投入上作出一定的平衡。要对现有的业务状况、未来的发展潜力以及爆发力上作出一些判断，很好的驾驭它是需要花费一些精力的，需要经验和技术能力同时发挥作用，让效率和投入达到最好的产出。

使用wrc工具进行压测：

命令：wrk -c 30 -t 10 -d 2s --latency http://www.baidu.com

参数释义：

-t：需要模拟的线程数

-c：需要模拟的连接数

-d：测试的持续时间

—-timeout 或 -T：超时的时间

–latency：显示延迟统计

-s 或 –script: lua脚本,使用方法往下看

-H, –header: 添加http header, 比如. “User-Agent: wrk”

二叉树的最小权值

发表于 2021-09-06

动态规划

W(v) = 1 + 2W(L) + 3W(R) + (C(L)) ^ 2 x C(R)。

设该树节点的个数为i

则 left + right = i - 1

dp[i]表示节点为i时，该树权值的最小值。

dp[i] = min(dp[i], 1 + 2 * dp[left] + 3 * dp[right] + left * left * right)

其中dp[0] = 0 dp[1] = 1

限流

漏桶算法

以恒定的速率进行传输数据，强行限制数据的传输速率。

令牌桶算法

从桶里面取令牌，如果没有令牌了则拒绝请求

使用场景

  • 漏桶算法可以保护第三方的系统，因为当我们调用第三方系统时，即使我们有能力处理这些并发请求，但是我们不能保证第三方的系统也有能力处理这些并发，但是我们没有办法去更改别人的系统，所以可以使用此算法保护别人的系统
  • 令牌桶算法可以有效的提高数据的传输能力，可以保护我们自己的系统。
